# Aliyun CDN Certificate Sync

## Purpose

This project automatically uploads TLS certificates generated by Kubernetes cert-manager to Aliyun Certificate Service and binds the uploaded certificate to one or more Aliyun CDN domains.

## Project Scope

The service is responsible for:

- Reading certificate and private key material from Kubernetes Secrets.
- Detecting certificate changes and expiration windows.
- Uploading or replacing certificates in Aliyun.
- Updating CDN domain certificate bindings safely and idempotently.
- Emitting logs and metrics for sync attempts, failures, and retries.

Out of scope:

- Issuing certificates (handled by cert-manager).
- Managing DNS records or CDN domain creation.
- Acting as a general-purpose secret synchronization service.

## Conventions

- Language: Go.
- Keep cloud provider integrations isolated in dedicated packages.
- Keep reconciliation logic deterministic and retry-safe.
- Avoid logging secret values (private key, full cert body).
- All behavior changes should go through OpenSpec proposals.

## Capabilities

Initial capabilities maintained under `openspec/specs/`:

- `certificate-sync`: certificate retrieval, comparison, and upload flow.
- `cdn-binding`: associating uploaded certificates with CDN domains.
- `runtime-config`: configuration, credentials, and safety defaults.

## Validation

If OpenSpec CLI is available, run strict validation before merging:

- `openspec list`
- `openspec list --specs`
- `openspec validate --strict`
